SMARTPAY CRM v3.3.1 — ПАРТНЕРЫ И РАЗГРАНИЧЕНИЕ ДОСТУПА

Что добавлено:
- новая вкладка «Партнеры» под вкладкой «Мерчанты»;
- отдельный лист Partners в Google Sheets;
- добавление и редактирование партнеров;
- выбор партнера при создании/редактировании мерчанта;
- фильтр мерчантов по партнеру;
- карточка партнера со списком привлеченных мерчантов;
- аналитика: всего и активных партнеров, количество привлеченных мерчантов, активные мерчанты, договоры, оборот и инвойсы;
- данные договора партнера, контактное лицо, менеджер, статус и условия вознаграждения;
- партнер отображается в общем списке и карточке мерчанта только у руководящего состава;
- партнер добавлен в экспорт отчета только для руководящего состава;
- поддержка мобильной версии и автоматического обновления данных.

Разграничение доступа:
- уровень «Руководство» видит вкладку «Партнеры», партнерскую аналитику, фильтр по партнерам и привязку партнера в карточке мерчанта;
- уровень «Менеджер» не видит вкладку «Партнеры», партнерские поля, фильтры и партнерскую историю;
- менеджеры продолжают видеть ВСЕХ мерчантов в общей вкладке «Мерчанты»;
- при редактировании мерчанта менеджером существующая привязка к партнеру не удаляется и не изменяется;
- backend не возвращает менеджерам список партнеров и не разрешает сохранять изменения партнеров.

Управление доступом:
1. Войдите под umed@smartpay.tj.
2. Откройте «Настройки» → «Пользователи».
3. Формат строки пользователя:
   логин; ФИО; должность; телефон; доступ
4. В последнем поле укажите «Руководство» или «Менеджер».
5. После изменения уровня доступа пользователю нужно выйти и войти заново.

Порядок установки:
1. В Google Apps Script замените действующий код содержимым файла Code.gs.
2. Сохраните проект и обновите действующее развертывание Web App:
   Deploy → Manage deployments → Edit → New version → Deploy.
3. Загрузите index.html, manifest.json и sw.js на GitHub/хостинг вместо текущих файлов.
4. Откройте CRM и обновите страницу. При необходимости выполните жесткое обновление Ctrl+F5.
5. Лист Partners и новые колонки в листе «Главная» создадутся автоматически при первой загрузке данных.

Важно:
- существующие мерчанты, задачи, история, договоры и текущая логика не удаляются;
- привязка партнера к мерчанту необязательна;
- старые мерчанты останутся без партнера, пока руководство не выберет его вручную;
- при переименовании партнера его название автоматически обновляется у связанных мерчантов;
- менеджеры видят весь общий список мерчантов, а не только назначенных на них.
