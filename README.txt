SMARTPAY CRM v3.6.1 — FAST CHAT

УСКОРЕНО
- Чат больше не загружает всю историю каждую секунду.
- Клиент получает только новые сообщения после последнего курсора.
- Новое сообщение временно попадает в быстрый Script Cache и быстрее доставляется получателю.
- При отсутствии новых сообщений сервер отвечает без чтения всего листа Chat.
- Отправленное сообщение сразу заменяется точной серверной записью без ожидания полной синхронизации.
- Чат не перерисовывается каждую секунду, если новых сообщений нет.
- Проверка при открытом чате выполняется примерно каждые 0,45 секунды; при закрытом — каждые 1,2 секунды.
- Сохранены шифрование, галочки прочтения, смайлики, звук и счетчики непрочитанных.

ВАЖНО: В ЭТОЙ ВЕРСИИ НУЖНО ОБНОВИТЬ И FRONTEND, И CODE.GS

УСТАНОВКА FRONTEND
1. В GitHub замените index.html, manifest.json и sw.js.
2. Нажмите Commit changes и дождитесь GitHub Pages deployment.

УСТАНОВКА BACKEND
1. Откройте Google Apps Script проекта CRM.
2. Полностью замените содержимое Code.gs файлом Code.gs из архива.
3. Нажмите «Развернуть» → «Управление развертываниями».
4. Откройте текущее развертывание, нажмите карандаш и выберите «Новая версия».
5. Нажмите «Развернуть». URL веб-приложения менять в index.html не нужно.

ОБНОВЛЕНИЕ У СОТРУДНИКОВ
1. Откройте /index.html?v=361.
2. Нажмите Ctrl+F5 или Ctrl+Shift+R.
3. При необходимости удалите старый Service Worker и очистите данные сайта.

ПРОВЕРКА
- В левом нижнем углу должно быть: v3.6.1 FAST CHAT.
- Откройте CRM у двух сотрудников одновременно.
- Сообщение должно появляться у получателя обычно менее чем за 1–2 секунды.
- Старые функции CRM и чата должны работать без изменений.

БЕЗОПАСНОСТЬ
- Code.gs содержит Telegram Bot Token. Не загружайте Code.gs в публичный GitHub-репозиторий.
