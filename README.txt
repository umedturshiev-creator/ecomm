SMARTPAY CRM v3.5.6 — READ RECEIPTS + EMOJI

ДОБАВЛЕНО
- В личном чате рядом с исходящим сообщением отображается одна галочка после успешной отправки.
- После открытия переписки получателем статус автоматически меняется на две галочки.
- Служебные подтверждения прочтения передаются через существующий защищенный личный чат и не показываются в переписке.
- Добавлена кнопка со смайликом возле поля ввода.
- Добавлен набор популярных эмоций, жестов, сердечек и рабочих символов.
- Смайлик вставляется в текущую позицию курсора, можно выбирать несколько подряд.
- Сохранены быстрая отправка сообщений и окно пароля поверх чата.

ВАЖНО
- Две галочки работают в личных чатах после того, как получатель открыл именно эту переписку.
- Для корректной работы подтверждений прочтения все сотрудники должны получить версию CRM v3.5.6.
- Code.gs менять и повторно развертывать не требуется.

УСТАНОВКА
1. В корне GitHub-репозитория замените:
   - index.html
   - manifest.json
   - sw.js
2. Нажмите Commit changes и дождитесь GitHub Pages deployment.
3. Откройте сайт по адресу:
   /index.html?v=356
4. Нажмите Ctrl+F5 или Ctrl+Shift+R.
5. Если старая версия осталась в кеше:
   F12 → Application → Service Workers → Unregister,
   затем Storage → Clear site data.

ПРОВЕРКА
- В левом нижнем углу: v3.5.6 READ RECEIPTS + EMOJI.
- Отправьте личное сообщение: сначала должна появиться одна галочка.
- Откройте эту переписку под учетной записью получателя.
- У отправителя после синхронизации должны появиться две галочки.
- Нажмите кнопку 😊 и выберите несколько смайликов.
