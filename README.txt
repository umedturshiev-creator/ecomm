SMARTPAY CRM v3.6.2 — MERCHANT FUNNEL

ДОБАВЛЕНО
- На дашборде появился отдельный блок «Воронка подключения мерчантов».
- Этапы: Потенциальный → В процессе → Договор → Интеграция → Тест → Активный.
- Для каждого этапа отображаются:
  • текущее количество мерчантов;
  • среднее количество дней на этапе;
  • сколько мерчантов уже перешло дальше;
  • процент перехода на следующие этапы.
- CRM автоматически определяет основную точку остановки и показывает ее под воронкой.
- Каждый этап кликабельный и открывает соответствующий список мерчантов.
- Добавлены отдельные статусы «Потенциальный», «Договор», «Интеграция» и «Тест».
- Старый статус «На интеграции и тест» сохранен и учитывается как этап «Интеграция».

УЧЕТ ВРЕМЕНИ НА ЭТАПЕ
- В таблицу «Главная» автоматически добавляются служебные колонки funnelStage и stageEnteredAt.
- При первом запуске CRM заполнит их для существующих мерчантов по доступной истории, дате договора, createdAt и updatedAt.
- При дальнейшем изменении этапа дата входа на этап фиксируется автоматически.
- Служебные поля не нужно редактировать вручную.

СОХРАНЕНО
- Ускоренная доставка чата.
- Звуковые уведомления, счетчики непрочитанных, одна и две галочки.
- Смайлики, личные сообщения без повторного ввода пароля.
- Все функции мерчантов, задач, партнеров, аналитики и резервного копирования.

ВАЖНО: НУЖНО ОБНОВИТЬ И FRONTEND, И CODE.GS

УСТАНОВКА FRONTEND
1. В GitHub замените index.html, manifest.json и sw.js.
2. Нажмите Commit changes и дождитесь GitHub Pages deployment.

УСТАНОВКА BACKEND
1. Откройте Google Apps Script проекта CRM.
2. Полностью замените содержимое Code.gs файлом Code.gs из архива.
3. Нажмите «Развернуть» → «Управление развертываниями».
4. Откройте текущее развертывание, нажмите карандаш и выберите «Новая версия».
5. Нажмите «Развернуть». URL веб-приложения менять не нужно.

ОБНОВЛЕНИЕ У СОТРУДНИКОВ
1. Откройте /index.html?v=362.
2. Нажмите Ctrl+F5 или Ctrl+Shift+R.
3. При необходимости удалите старый Service Worker и очистите данные сайта.

ПРОВЕРКА
- В левом нижнем углу должно быть: v3.6.2 MERCHANT FUNNEL.
- На дашборде под верхними карточками должна отображаться воронка из шести этапов.
- Нажатие на этап должно открывать отфильтрованный список мерчантов.
- После изменения статуса мерчанта его этап и время на этапе должны обновиться.

ДЛЯ ТОЧНОЙ ВОРОНКИ
- Используйте отдельный статус «Интеграция», когда идет подключение.
- После перехода к проверочным платежам меняйте статус на «Тест».
- После запуска меняйте статус на «Активный».

БЕЗОПАСНОСТЬ
- Code.gs содержит Telegram Bot Token. Не загружайте Code.gs в публичный GitHub-репозиторий.
